# Vibecoder Trial Task — Development & Deployment Requirements

## Overview

Build a working prototype of an analytics dashboard within 48 hours.

The goal is to demonstrate:
- Fast execution
- Clean UI/UX
- Independent problem solving
- Good communication skills
- Ability to ship quickly

---

# Required Tech Stack

You must use:

- React
- TypeScript
- Tailwind CSS

You may use any charting library you prefer, such as:
- Recharts
- Chart.js
- ApexCharts
- ECharts

---

# Dashboard Requirements

Create 3 main sections:

## 1. YouTube Channel Metrics

Display:
- Views
- Revenue

---

## 2. Web Store Metrics

Display:
- Units Sold
- Revenue
- Conversion Rate

---

## 3. Combined Analytics

Display combined analytics from:
- YouTube channels
- Web stores

The UI should make it easy to:
- Compare channels/stores
- View individual performance
- Understand overall metrics quickly

---

# Data Requirements

You may use mock/fake data.

You do NOT need:
- Real APIs
- Backend database
- Authentication

You can invent:
- Channel names
- Store names
- Metrics data

---

# UI/UX Expectations

The dashboard should:
- Be visually clean
- Be easy to understand
- Present analytics clearly
- Prioritize usability over complexity

Perfection is NOT required.

The priority is:
- Speed
- Clarity
- Execution

---

Optional ideas to improve presentation:
- KPI cards
